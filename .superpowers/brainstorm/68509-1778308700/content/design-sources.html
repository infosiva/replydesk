Which design source strategy fits best?

We have 15+ projects. Different sources = different personalities. Pick the approach.

A

Assign by project type

Dashboards → Tremor/shadcn. Landing/games → Magic UI + Aceternity. Compliance/pro tools → shadcn clean. Each project gets the right tool for its personality.

Pros

  • Purposeful variety
  • Best tool per job

Cons

  • More libraries to learn
B

One shared system, per-project themes

shadcn/ui as base everywhere. Magic UI animations layered on top. Theme token per project (color, radius, font). Consistent but still distinct.

Pros

  • One codebase pattern
  • Easy to maintain

Cons

  • Less personality difference
C

3-tier pipeline: Stitch → Claude → Code

Google Stitch generates wireframe → MCP exports it → Claude /design-html renders it → convert to React. Every project starts from a real wireframe, not guesswork.

Pros

  • Most intentional designs
  • Fits Initiative #2

Cons

  • Needs Stitch MCP built first
