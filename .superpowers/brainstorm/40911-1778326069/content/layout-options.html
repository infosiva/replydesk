Analytics Dashboard — Where should it live?

tracker-api already has all the data at port 3098. Pick the home for the dashboard UI.

A

Pure HTML in tracker-api/public

Single HTML file. Deploy = scp to VPS. Zero framework. Live in minutes. No build step.

Pros

  • Instant — no deploy pipeline
  • Reads /stats directly on same host
  • Lives forever on VPS port 3098

Cons

  • No auth (behind VPN/IP only)
  • Raw HTML/CSS — harder to extend
B

New page in agenttrace (agentlogs.app)

Add /dashboard route to existing Next.js app. Vercel deploy. Proper React components. Uses Tremor (already in design system for agenttrace).

Pros

  • Proper UI with Tremor charts
  • Public URL (agentlogs.app/dashboard)
  • Auth can be added later

Cons

  • Needs Vercel deploy + CORS on tracker-api
  • Slower to ship
C

Extend site-watchdog dashboard (port 3099)

Site-watchdog already shows uptime + Vercel deploy status. Add analytics tab to existing Node/Express dashboard.

Pros

  • Unified ops view — uptime + analytics together
  • Already on VPS, scp deploy
  • Same codebase as site-watchdog

Cons

  • site-watchdog is Node/TS — more complex than option A
  • UI is already busy
